# Комплексирование ИНС и ГНСС (суммарная навигационная система)

Реализация алгоритма объединения инерциальной навигационной системы (ИНС) и глобальной навигационной спутниковой системы (ГНСС) в единую суммарную систему на основе **фильтра Калмана**.

## Теоретическая основа (исходные данные)

Основной навигационной системой многих современных роботизированных объектов (самолёты, беспилотные воздушные аппараты, подводные аппараты различных классов, беспилотный автомобильный транспорт и т.д.) по причине высокой степени надёжности и автономности является **бесплатформенная инерциальная навигационная система (БИНС)**. Основной недостаток БИНС — быстрое накопление ошибок, поэтому БИНС требует периодической или постоянной корректировки (исключение — объекты с коротким сроком активной эксплуатации).

Смысл построения **комплексных навигационных систем** — объединение датчиков и систем, основанных на различных физических принципах, для оценки и корректировки погрешностей отдельных систем. Традиционные принципы:

1. **БИНС — «ядро»** комплексной навигационной системы; остальные навигационные датчики и системы применяются для коррекции БИНС.
2. **Основной корректор БИНС — приёмник ГНСС**, за исключением случаев, когда использование спутниковых систем невозможно или затруднительно (роботы в помещениях, подводные роботы, космические системы и т.д.).
3. Для оценки погрешностей БИНС используется **алгоритм оптимального фильтра Калмана (ОФК)**, его модификации или схожие алгоритмы.

В проекте реализована относительно простая структурная схема алгоритма комплексной обработки на основе ОФК: ИНС даёт прогноз, ГНСС — измерения для коррекции.

## Описание

- **ИНС** даёт высокочастотную оценку положения и скорости путём интегрирования показаний акселерометров и гироскопов, но со временем накапливает ошибку (дрейф).
- **ГНСС** даёт абсолютные измерения положения (и скорости) с меньшей частотой и возможными срывами сигнала, но без дрейфа.

Комплексирование позволяет:
- получать непрерывную оценку траектории с частотой ИНС;
- корректировать дрейф ИНС по измерениям ГНСС;
- сохранять приемлемую точность при кратковременных срывах ГНСС за счёт прогноза по ИНС.

## Алгоритм

1. **Состояние** фильтра Калмана: вектор положения и скорости в навигационной системе координат (НЭЗ):  
   **x** = [p_x, p_y, p_z, v_x, v_y, v_z] (м, м/с в ENU).

2. **Прогноз (ИНС):** экстраполяция по линейной модели с постоянным ускорением за шаг, вход — ускорение от ИНС в ENU. Учитывается шум процесса (ошибки ИНС).

3. **Коррекция (ГНСС):** при поступлении измерений ГНСС (положение и при необходимости скорость) выполняется шаг обновления фильтра Калмана.

4. На выходе — оценка положения и скорости суммарной системы и ковариация ошибки.

## Алгоритм и конвенции

**Поток данных:** сырые ИМУ (ax, ay, az, gx, gy, gz) → перевод в навигационную СК (body→nav по ориентации с гироскопа) → ускорение в НЭЗ → фильтр Калмана (прогноз по ускорению, коррекция по ГНСС в ENU) → положение и скорость в ENU → при необходимости перевод в широту/долготу для карты.

**Системы координат и единицы:**
- **Навигационная СК (НЭЗ):** ENU — East, North, Up. Состояние фильтра и все позиции/скорости в метрах: `[East, North, Up]`, индексы 0, 1, 2.
- **Связанная СК (body):** X вперёд, Y вправо, Z вниз. Курс (yaw) отсчитывается от направления на North по часовой стрелке; в момент старта курс = 0, далее интегрируется по гироскопу.

**Переменный шаг по времени:** при неравномерной сетке (например, данные с Arduino) в `predict(acceleration, dt=...)` передаётся фактический шаг `dt` между выборками; матрицы прогноза пересчитываются под этот шаг.

**Пропуски ИМУ:** при отсутствии или невалидных измерениях акселерометра/гироскопа (NaN) в пайплайне проверки CSV подставляется нулевое ускорение, чтобы фильтр продолжал работать.

**Единицы в CSV:** колонки `ax,ay,az,gx,gy,gz` могут быть в **сырых LSB** (как с MPU6050) или уже в **м/с² и град/с**. Скрипты автоматически определяют единицы по диапазону (большие числа → LSB) и применяют масштабы MPU6050 при необходимости. Формат выдачи с платы (Adafruit и др.): см. раздел «Формат CSV» ниже.

## Формат CSV (запись с платы)

Строка каждые **100 мс**:

| Колонка | Значение | Единицы / примечания |
|---------|----------|----------------------|
| `time_ms` | `millis()` | Время от старта, не UTC |
| `lat`, `lon` | Широта, долгота | Градусы; при невалидном ГНСС — `nan` |
| `alt_m` | Высота | Метры |
| `speed_mps` | Скорость | м/с |
| `sats` | Число спутников | При невалидном ГНСС → 0 |
| `fix` | Признак фиксации | 1 если `gps.location.isValid()`, иначе 0 |
| `ax`, `ay`, `az` | Ускорение | **м/с²** (как отдаёт Adafruit) |
| `gx`, `gy`, `gz` | Угловые скорости | **град/с** (из рад/с: ×180/π) |

Числа печатаются через `printFloat()`: либо `nan`, либо число с 6 знаками после запятой.

### Защита и диагностика

При загрузке CSV скрипты проверяют:
- **Шаг по времени** — ожидается ~100 мс; при сильном отклонении выдаётся предупреждение.
- **ГНСС:** доля строк с валидными `lat`/`lon`, количество строк с `fix=1`; при отсутствии валидных координат расчёт не запускается.
- **ИМУ:** диапазоны `ax,ay,az` и `gx,gy,gz`; по ним определяется, в LSB или в физических единицах записаны данные.
- **Пропуски:** строки с `nan` в акселерометре/гироскопе при обработке заменяются нулевым ускорением (фильтр не падает).

Полный отчёт диагностики выводит `analyze_ins_csv.py`.

## Связанные оси и ориентация по гироскопу

Оси датчиков **связаны с корпусом** (body). Курс в момент запуска = 0, далее крен, тангаж и курс интегрируются по гироскопу. По текущей ориентации показания акселерометра переводятся из связанной СК в **навигационную (ENU)**; полученное ускорение подаётся в фильтр Калмана.

- **С Arduino нужны только чистые (сырые) значения:** акселерометр `ax, ay, az` и гироскоп `gx, gy, gz` в LSB. Масштабирование и перевод в нав. СК делаются в Python.
- Модуль `imu_body_to_nav.py`: инициализация курса = 0, обновление ориентации по гироскопу, перевод ускорения body → nav. Поддержка масштабов для MPU6050 и своих калибровок.

## Файлы

- `ins_gnss_filter.py` — класс фильтра Калмана и функция запуска комплексирования на траектории.
- `imu_body_to_nav.py` — связанные оси: курс = 0, ориентация по гироскопу, перевод ускорения в нав. СК; масштабы для сырых значений Arduino.
- `arduino_imu_reader.py` — чтение с Arduino строк вида `ax,ay,az,gx,gy,gz` (сырые), вызов перевода в нав. СК и при необходимости передача в фильтр Калмана.
- `arduino_imu_raw/arduino_imu_raw.ino` — скетч для Arduino: вывод только сырых значений акселерометра и гироскопа (MPU6050 по I2C).
- `analyze_ins_csv.py` — **анализ исходного CSV**: определение единиц ИМУ (LSB или м/с², град/с), полный пересчёт ускорения в нав. СК, интеграция «только ИНС» и ИНС+ГНСС, вывод причин плохой работы ИНС (дрейф, единицы, bias). Запуск: `python analyze_ins_csv.py путь.csv`
- `simulate_trajectory.py` — генерация тестовой траектории, имитация ИНС/ГНСС (в т.ч. в связанных осях) и построение графиков и карты.
- `geo_map.py` — привязка к карте: перевод координат (ENU → широта/долгота) и построение интерактивной карты.
- `requirements.txt` — зависимости (numpy, matplotlib, folium).

## Карта: вход координаты — выход положение на карте

**Вход:** опорные координаты (широта, долгота, высота) — точка привязки траектории к карте.  
**Выход:** интерактивная карта (OpenStreetMap) с траекториями: истина, только ИНС, суммарная (ИНС+ГНСС), ГНСС.

Запуск с координатами по умолчанию (Москва):

```bash
pip install -r requirements.txt
python simulate_trajectory.py
```

Свои координаты (широта, долгота, высота):

```bash
python simulate_trajectory.py --lat 55.7558 --lon 37.6173 --alt 150
```

Создаются файлы `ins_gnss_result.png` (графики) и `ins_gnss_map.html` (карта — откройте в браузере).

## Использование в коде

```python
from ins_gnss_filter import FilterConfig, INSGNSSKalmanFilter, run_ins_gnss_complex
import numpy as np

config = FilterConfig(
    dt=0.1,
    sigma_acc=0.15,
    sigma_gnss_pos=1.5,
    sigma_gnss_vel=0.2,
    use_gnss_velocity=True,
)
x0 = np.array([0, 0, 0, 1, 0, 0])  # начальное положение и скорость
kf = INSGNSSKalmanFilter(config, x0)

# В цикле по времени:
kf.predict(acceleration_ins)   # ускорение от ИНС [ax, ay, az]
kf.update(position_gnss)      # или [px, py, pz, vx, vy, vz] при use_gnss_velocity=True
pos = kf.get_position()
vel = kf.get_velocity()
```

Или на всей траектории сразу:

```python
positions, velocities, covariances = run_ins_gnss_complex(
    time_grid=t, acc_ins=acc_ins, pos_gnss=gnss_pos, vel_gnss=gnss_vel, config=config
)
```

### Работа с Arduino (сырые акселерометр + гироскоп)

```python
from imu_body_to_nav import ArduinoIMUScales, BodyToNavProcessor
from arduino_imu_reader import parse_arduino_imu_line, read_arduino_imu_loop
import serial

scales = ArduinoIMUScales()
with serial.Serial("COM3", 115200) as ser:
    def on_nav(accel_nav, roll, pitch, yaw, gyro_body):
        # accel_nav — м/с² в НЭЗ; gyro_body — (3,) рад/с в связанной СК
        kf.predict(accel_nav)
        print("gyro wx, wy, wz:", gyro_body)
    read_arduino_imu_loop(ser, dt=0.01, scales=scales, on_nav_accel=on_nav)
```

В симуляции при `use_body_frame=True` создаётся файл `ins_gnss_imu_log.csv` с колонками: время, позиция, скорость, ускорение в нав. СК, **гироскоп (wx, wy, wz) в рад/с**.
