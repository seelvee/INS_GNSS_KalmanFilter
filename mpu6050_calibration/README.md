# Калибровка MPU6050 для ИНС / фильтра Калмана

Проект из двух частей:

1. **Arduino Uno** — прошивка `arduino/mpu6050_stream/mpu6050_stream.ino`: только **Wire.h**, обмен по UART (**115200**), поток сырых отсчётов MPU6050 и команды `START_STREAM` / `STOP_STREAM` / `WHOAMI` / `RESET`.
2. **Python 3** — скрипты в каталоге `python/`: сбор данных с COM-порта, расчёт параметров, графики, сохранение `raw_log.csv`, `calibration_report.csv`, `imu_calibration.json`.

## Подключение аппаратуры

| Сигнал | Arduino Uno |
|--------|-------------|
| VCC    | **5 V или 3,3 V** — как указано для вашего модуля MPU6050 |
| GND    | GND |
| SDA    | **A4** |
| SCL    | **A5** |

AD0 на плате модуля обычно к **GND** (адрес **0x68**). Если AD0 на **VCC**, используйте **0x69** и измените константу `MPU6050_ADDR` в скетче.

## Загрузка скетча

1. Установите [Arduino IDE](https://www.arduino.cc/en/software).
2. Откройте файл  
   `mpu6050_calibration/arduino/mpu6050_stream/mpu6050_stream.ino`  
   (IDE создаст папку скетча `mpu6050_stream` автоматически, если вы открываете именно `.ino`).
3. Платформа: **Arduino Uno**, COM-порт — ваш кабель USB.
4. Загрузите прошивку.
5. Монитор порта **115200** (перевод строки **LF** или «NL»).

После старта в мониторе появятся строки `# ...`. Пока не отправите **`START_STREAM`**, строк CSV не будет — это нормально.

Формат одной строки потока:

```text
time_ms,ax_raw,ay_raw,az_raw,gx_raw,gy_raw,gz_raw,temp_raw
```

Все значения — **целые сырые 16-битные коды** из регистров (аксель от `0x3B`, температура `0x41`, гиро от `0x43`). Шкала прошивки: **±2g**, **±250 °/с**, частота порядка **100 Гц**.

Команды (одна строка в монитор):

| Команда       | Действие                          |
|---------------|-----------------------------------|
| `START_STREAM`| начать вывод CSV                  |
| `STOP_STREAM` | остановить вывод                  |
| `WHOAMI`      | прочитать регистр WHO_AM_I        |
| `RESET`       | сброс и переинициализация MPU6050 |

Python перед сбором сам отправляет `STOP_STREAM` / `START_STREAM`, монитор порта лучше **закрыть**.

## Запуск: режим stationary (неподвижный модуль)

Снимает данные за заданное время (например **300 с**) для оценки:

- средних и стандартных отклонений сырых каналов;
- **bias гироскопа** по среднему `gx_raw, gy_raw, gz_raw` → перевод в **рад/с**;
- шум гироскопа: `std(g - bias)` в **рад/с**, ковариация **3×3**;
- для акселерометра в м/с²: перевод по номиналу **16384 LSB/g**, **g = 9.80665** м/с²; по каждой оси вычитается **среднее** (постоянная гравитация и горизонтальное положение — не считаются шумом), шум = **`std(signal - mean)`**, ковариация шума **3×3** в связанной СК датчика.

Пример:

```bash
python calibrate_mpu6050.py --port COM5 --baud 115200 --mode stationary --duration 300 --outdir ../output
```

Результаты по умолчанию в каталоге `output/` (можно изменить `--outdir`):

- `raw_log.csv`
- `calibration_report.csv`
- `imu_calibration.json`
- `plots/stationary_*.png`

## Запуск: режим six_position (калибровка акселерометра)

Последовательно запрашиваются положения: **+X**, **−X**, **+Y**, **−Y**, **+Z**, **−Z** «вверх» относительно вертикали к столу. В каждом положении после **Enter** снимается поток (**по умолчанию 20 с** на позу).

Для каждой оси (обработка в **сырых кодах**):

```text
bias_axis = (mean_plus + mean_minus) / 2
scale_axis = 2 * g / (mean_plus - mean_minus)    ; единицы: (м/с²)/LSB
```

Ускорение в м/с²:

```text
a_i = (raw_i - bias_i) * scale_i
```

После расчёта выводится проверка: модуль вектора `√(ax²+ay²+az²)` в каждой позе должен быть близок к **9.80665** м/с².

Пример:

```bash
python calibrate_mpu6050.py --port COM5 --mode six_position --pose-duration 20 --outdir ../output
```

В этом режиме поля **`gyro_bias_*`** и матрицы шума гироскопа в JSON могут быть **`null`** — их нужно получить отдельным запуском **stationary** и при необходимости объединить в отчёте вручную.

Графики: временные ряды сохраняются как `plots/sixpos_accel_time.png`, `plots/sixpos_gyro_time.png`. Гистограммы шума строятся для режима **stationary**.

## Файл `imu_calibration.json`

Ключевые поля:

- **`accel_bias_raw`**, **`accel_scale`** — калибровка акселя (six_position или номинальный шкала только для stationary).
- **`gyro_bias_raw`**, **`gyro_bias_rad_s`** — смещение гироскопа (после stationary).
- **`accel_noise_std_mps2`**, **`gyro_noise_std_rad_s`**, **`accel_noise_cov_mps2`**, **`gyro_noise_cov_rad_s`** — оценки шума (обычно из stationary).

Поле **`accel_bias_mps2`** хранит **bias_raw × scale** для альтернативной записи  
`a = raw * scale - accel_bias_mps2`; основная рекомендуемая форма указана в **`accel_model_note`**.

### Важно про физику (для отчёта)

1. **Bias акселерометра и шум** — разные вещи: bias систематический сдвиг (компенсируется калибровкой положений); шум — случайный разброс вокруг **вычитания среднего** при статике.
2. Акселерометр измеряет **удельную силу** (реакцию опоры с учётом гравитации), а не «чистое кинематическое ускорение» в инерциальной системе без ориентации.
3. Чтобы перейти в **навигационную** СК (NED/ENU и т.д.), нужна **ориентация** (углы/кватернион) и пересчёт вектора силы тяжести и ускорений.
4. Для **фильтра Калмана**: дисперсии шума процесса **Q** часто связывают с ковариациями возмущений состояния (например из **`accel_noise_cov_mps2`** и **`gyro_noise_cov_rad_s`** после перевода в ваш дискретный шаг **dt** и модель **F**, **G**). Дисперсии измерений ГНСС задаются в **R** и не смешиваются с шумом процесса ИМУ.

### Использование ковариаций для **Q**

Квадрат корня диагонали **`accel_noise_cov_mps2`** и **`gyro_noise_cov_rad_s`** задаёт порядок стандартных отклонений шума акселя и гиро в покое. В дискретной модели элементы **Q** получают умножением на функции **dt** и матрицы возмущений **G** (зависят от выбранного вектора состояния). Имеет смысл явно записать в отчёте: «ковариации оценены по статическому логу, дискретизация …».

## Обработка ошибок (Python)

Скрипт сообщает об ошибках при:

- недоступности COM-порта;
- таймауте старта потока или длительном отсутствии строк (**обрыв связи**);
- слишком малом числе валидных строк;
- неверном формате строки CSV.

При ошибке отправляется **`STOP_STREAM`**.
